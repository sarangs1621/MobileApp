# M7 — Report Cards & Academic Results — ✅ STEPS 1–10 COMPLETE (awaiting approval)

Scope (per ADR-014): **`ReportCard` — Enrollment-owned, `kind`-discriminated
(EXAM/TERM/ANNUAL) with nullable `examId`/`termId` scope; lifecycle
DRAFT→SUBMITTED→APPROVED→PUBLISHED (+ SUPERSEDED/REVOKED); snapshot (attendance %,
rank, GPA) frozen at APPROVE; correction = new version (supersede-then-publish, one
tx); class-teacher-authored remark via ADR-015; stored-PDF `pdfPath` (private bucket,
ADR-004).** The academic **reporting layer** — consumes M3 Enrollment, M4 Attendance,
M5 Marks/GPA read-only; owns only report-specific fields. **ADR-014 is the source of
truth** (extends ADR-009's `examId`-nullable seam; the discriminator + per-kind
uniqueness it deferred are realized here). Locked product decisions R1/R2/R3: class
teacher drafts remark + submits, office/principal approve/publish/reopen/revoke (R1);
rank stored + own-rank-only, app-layer hide (R2); published immutable, corrections
versioned, prior retained, every publish/correction audited (R3). Out: PDF rendering,
notifications, CGPA-across-years — later milestones.

Roles: SUPER_ADMIN + OFFICE_ADMIN full lifecycle school-wide (`report_card:manage`);
TEACHER drafts remark + submits for **own class-teacher section** (`report_card:remark`,
narrowed by `assertClassTeacherOfEnrollment` — a subject teacher of the section is
refused); PARENT reads own child's **PUBLISHED** cards (`report_card:read`); ACCOUNTANT
none. **No Student login — the Parent is the reader.**

## Deviations & known limitations

- **APPROVED ≡ ADR-014 §2's "GENERATED"** — renamed to match R1's "approve" verb; the
  approval gap is a real state (office/principal review before release).
- **Correction = option B (supersession/versioning)** — chosen over ADR-014's §4-recommended
  (A) overwrite-and-audit, because R3 requires an immutable retained history of every version.
- **Approve requires SUBMITTED** (approve-from-DRAFT removed in the Step-5 review) — every card
  passes the class-teacher submit gate; a section with no class teacher has no submitter.
- **No PDF generation** — `pdfPath` + a private bucket are provisioned; rendering + bilingual
  (en+ml) templates deferred. **No report-card notifications**; **no CGPA-across-years** (reserved).
- **Mobile = parent read only**; **web card list = current-year** (cross-year trail deferred).
- **Runbook-gated:** a private report-card bucket must be provisioned before live PDF generation
  (a one-time step, like `homework-files` in M6).
