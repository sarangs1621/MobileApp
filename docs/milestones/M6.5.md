# M6.5 — Class Teacher Management — ✅ STEPS 1–10 COMPLETE (awaiting approval)

Scope: **one domain concept — `Section → Class Teacher (for a year)`.** A dedicated
`ClassTeacherAssignment` model (**ADR-015**): the CURRENT holder of a
`(academicYear × section)` slot → exactly ONE teacher. It is **not** a flag on
`TeacherAssignment` (a teacher may teach several subjects in a section, and a class
teacher may teach none) — `TeacherAssignment` stays frozen. Introduced to supply the
architectural dependency report cards (M7 / ADR-014) need; **not** part of report cards,
attendance, homework, exams, timetable, or messaging.

Model: `ClassTeacherAssignment(id, schoolId, academicYearId, sectionId, teacherId→User,
assignedAt, createdByStaffId→Staff, createdAt, updatedAt)`; `@@unique(academicYearId, sectionId)`;
all FKs **Restrict**. `teacherId → User` (matches TeacherAssignment + RLS `auth.uid()`);
`createdByStaffId → Staff` (B3 audit actor); `assignedAt` = when the current teacher took
the slot (re-stamped on replace). Year-scoped → cross-year history is per-year rows;
within-year replacement is an **in-place update** (ADR-010 §5), never a 2nd row —
history lives in `AuditLog`. **No `endedAt`, no `isActive`, no partial index.**

Roles: SUPER_ADMIN / OFFICE_ADMIN manage (assign/replace/remove) under `academic:manage`;
TEACHER reads (`academic:read`, RLS SELECT own); PARENT / ACCOUNTANT none. No new permission
(reuses academic:manage / academic:read); frozen role→permission policy untouched.

## Architectural review (approved)

Replace mutates the row **in place** (Option A), it does not create history rows.
Evidence: `ClassTeacherAssignment` is the structural twin of `Enrollment` (year-scoped single
slot), whose within-year reassignment is an audited in-place update (ADR-010 §5); the only
consumer is a point-in-time gate (report-card remark authorship snapshots the author at write
time); no consumer requires historical class-teacher identity. `endedAt` is the documented
additive extension **iff** a real tenure-history requirement ever lands.

## Invariants (ADR-015)

One class teacher per section per year (DB unique) · replace = in-place update, never a 2nd row ·
`teacherId → User`, `createdByStaffId → Staff` · Restrict everywhere (history preserved) ·
`assignedAt` = current holder's start (re-stamped on replace; prior value in AuditLog) ·
every mutation audited in-tx · TeacherAssignment frozen · no `isClassTeacher`, no `endedAt`,
no `isActive`, no temporal tables.
