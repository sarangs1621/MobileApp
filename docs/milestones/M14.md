# M14 — Analytics & Reporting — ✅ STEPS 1–9 COMPLETE (awaiting approval)

Scope: **role-aware dashboards, charts, and CSV export** over frozen M1–M13 — every number **computed live, on demand**,
read-only. **Purely additive: zero schema change, no new table, no new permission, no snapshot, no cron, no cache, no
materialized view.** **ADR-022** governs the design.

> Analytics is aggregation of data the principal can **already read**, so each panel authorizes by **reusing the
> underlying domain read permission + scope** (no `analytics:read`; the flag-gated `analytics:view` is superseded). Over-
> exposure is structurally impossible — a parent's attendance panel passes the same `attendance:read + ownChild` gate as
> the raw feed.

## Permissions

**None added.** Reuses `attendance:read` / `marks:read` / `fee:read` / `report_card:read` / `behaviour:read` /
`student:read` — each with its existing scope (admin school / teacher own-section / parent own-child). School-wide panels
(`schoolSummary`, `feeCollection`, `topPerformers`, `atRiskStudents`) add an **admin-only guard**. The flag-gated,
SA-only `analytics:view` matrix row is **superseded** (as M9/M10 superseded their planned rows). **No feature flag.**

## Deviations / v1 simplifications (all flagged + approved at each STOP)

1. **New read-only aggregate methods added to repositories** — in-service JS aggregation duplicates the DB and doesn't
   scale; still additive + read-only, no schema.
2. **Recharts** is a net-new web dependency (the brief's named fallback; no chart lib existed).
3. **Mobile ships hand-rolled widgets, not charts** (no `react-native-svg` present) — upgradable on request.
4. **Exam trend = published report-card GPA snapshots**; no subject-averages / per-exam distribution.
5. **Admin school-wide attendance-trend & exam-distribution time series not built** (no series aggregate) — attendance is
   a KPI; admin charts are fee-collection (bar) + invoices-by-status (pie). Student-growth chart omitted (raw year-id
   labels).
6. **compute-on-read at all scopes**; school-wide top/at-risk are O(active-year enrollments) live — the named ADR §4
   ceiling; cached summary is the deferred upgrade.
7. **The two legacy `downloadCsv` copies left in place** — consolidating would touch frozen M9/M12 UI for no functional gain.
