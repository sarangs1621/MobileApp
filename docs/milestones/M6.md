# M6 — Homework & Assignment Management — ✅ STEPS 1–10 COMPLETE (awaiting approval)

Scope (per M6 kickoff brief): **Homework (Subject×Section, year-stamped),
HomeworkAttachment (teacher files), HomeworkSubmission (per Enrollment, unique),
SubmissionAttachment (append-only, attempt-tagged), HomeworkFeedback (immutable,
text-only), private `homework-files` bucket.** The brief **overrides the PRD's
distribution-only decision** (§8.6 / decision-#13): parent submissions, uploads,
teacher review, and feedback are **core** — no `homework-uploads` flag. Out:
notifications, chat/messaging, grading/marks on homework, plagiarism, report cards —
later milestones.

Roles: SUPER_ADMIN + OFFICE_ADMIN full management + review (school-wide); TEACHER
manages/reviews **own subject×section** (derived from `TeacherAssignment`); PARENT
reads own child's PUBLISHED/CLOSED homework (§10) + submits/resubmits for own child +
reads own feedback; ACCOUNTANT none. **No Student login — the Parent is the actor.**
**ADR-013 is the source of truth** (extends ADR-010/011/012 + ADR-004): submissions
key to **Enrollment, never Student**; homework **is** the register (no fan-out);
ownership **derives** from TeacherAssignment; publication = parent-visibility gate,
single grain; content frozen at publish, dueDate extend-only; every load-bearing
pattern is a reuse of M3–M5.

## Deviations & known limitations

- **Parent upload path** keyed by `{homeworkId}/{enrollmentId}/{attempt}` (not
  `{submissionId}` as ADR-013 §9 sketched) — enables single-transaction submit; same
  storage properties, no behavioural change.
- **Mobile file upload web-only** (download works on both surfaces).
- **OFFICE_ADMIN create-from-scratch web picker unbuilt** — `homework.targets` is
  teacher-scoped; the service supports admin create, only the all-subjects×all-sections
  picker is missing (admins manage/review/upload to existing homework).
- **No un-review correction**; **no homework notifications**; **storage byte-cleanup
  deferred** (append-only history).
- **Runbook-gated:** the real upload→download round-trip against a provisioned
  `homework-files` bucket is a one-time manual check (`RUNBOOK_SUPABASE_SETUP.md §3c`)
  — the only file path CI cannot exercise (no bucket in CI, same as M3 storage).
