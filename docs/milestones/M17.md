## Deviations (ADR-025, approved at Step 1)
1. `/api/health` stays liveness-only; DB+storage go in `/api/ready` (a liveness probe that fails on DB-down causes
   pointless restarts).
2. CSP ships **report-only** first (enforce phase = nonce wiring).
3. Backup/DR = runbooks over Supabase PITR/`pg_dump`, **not** a built backup service.
4. No orchestration platform prescribed (Docker + compose only).
5. `system:manage` is **Super-Admin-only** (narrower than M16's SA+OA `settings:manage`).
