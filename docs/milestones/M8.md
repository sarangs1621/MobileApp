## Step-7 consistency audit — accepted raw-id fallbacks

These `?? id` sites remain, **explicitly accepted**: all are frozen **M2/M4/M5** screens whose viewer always
holds the corresponding read permission (`academic:read` / `student:read`), so the client name-map is always
populated and the `?? id` is a defensive **loading** fallback that never persists as a label — and they are
outside M8's identity scope (no frozen-logic change permitted).

- `apps/mobile/(app)/attendance/sections.tsx` — `className ?? classId` (teacher/admin, `academic:read`)
- `apps/mobile/(app)/attendance/mark/[sectionId].tsx` — `studentName ?? studentId` (teacher, `student:read`)
- `apps/mobile/(app)/academic/assignments.tsx` — `sectionLabel ?? sectionId` (`academic:read`)
- `apps/mobile/(app)/exam/mark/[assessmentId].tsx` — `studentName ?? studentId` (teacher, `student:read`)
- `apps/web/(app)/attendance/summary` + `attendance/mark` — `studentName ?? studentId` (admin/teacher)
- `apps/web/(app)/academic/assignments` — `sectionLabel ?? sectionId` (`academic:read`)
- `apps/web/(app)/exams/[examId]` — `studentName ?? studentId` (admin/teacher); one is a CSV **filename** fallback
