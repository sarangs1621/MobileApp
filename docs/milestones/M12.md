# M12 — Student Discipline & Leave Management — ✅ STEPS 1–9 COMPLETE (awaiting approval)

Scope: **behaviour incidents / teacher referrals** over frozen M1–M11 (a longitudinal `BehaviourIncident` with an
`OPEN→IN_PROGRESS→RESOLVED→CLOSED` lifecycle, immutable after CLOSED), plus **parent-leave notifications** — the one
gap in the **already-shipped M4 leave workflow**. On create a behaviour incident **optionally** emits an M10
`Notification(type=BEHAVIOUR)`; a leave decision now emits `Notification(type=LEAVE)`. Purely **additive** (1 new
table, 3 enums, 2 `NotificationType` values, 3 permissions). **NO counselling / medical records / detention /
attendance auto-edit / push / SMS / WhatsApp.** **ADR-020** governs the design.

> **The leave half was already built in M4 (ADR-011) and is frozen.** M12 does not rebuild it — a second
> `LeaveRequest`/`LeaveStatus` is impossible (Prisma name collision) and forbidden (freeze/zero-drift). M12 reuses the
> M4 table/services/screens/permissions verbatim and adds only the notification-on-decision wrap.

## Permissions

`behaviour:manage` (SA/OA — any student + console) · `behaviour:record` (TEACHER — own-section students, `teacherId`=self)
· `behaviour:read` (SA/OA/T/P — business-scoped). **Leave reuses `leave:apply`/`leave:decide`/`leave:read` (frozen M4)
— no new leave permission.** **Permission-only — no feature flag.**

## Deviations (ADR-020)

1. **Leave is reused, not rebuilt** — M4's `LeaveRequest`/`LeaveStatus`/services/screens/permissions are frozen; a
   second table is impossible + freeze-forbidden. M12 adds only the notify-on-decide wrap; attachment + reviewRemark deferred.
2. **Two new `NotificationType` values** (`BEHAVIOUR`, `LEAVE`) — an `ALTER TYPE … ADD VALUE`, not a frozen-table ALTER.
3. **`BehaviourIncident` keeps both `studentId` + `enrollmentId`** — justified divergence from the ADR-011 attendance
   idiom (discipline is a longitudinal *person* record; attendance is per-session).
4. The leave-notify wrap **repoints the frozen M4 attendance router** to `decideLeaveAndNotify` (the disclosed ADR-018
   §3/#4 pattern; the service stays byte-identical).
5. **"Excused attendance linkage" is already M4** and the explicit "excused" op is deferred (the brief defers it) — M12
   adds no attendance write.
